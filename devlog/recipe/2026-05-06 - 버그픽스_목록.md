## [2026-05-06] 레시피 상세 조회에서 SELECT *로 미노출 컬럼(user_id 등)이 응답에 포함되는 문제

- **수정:** 명시적 컬럼 선택(`recipe_id, title, content, abv_range, main_ingredient, ai_sub_ingredient, target_flavor, concept, summary, author_type, status, is_fundable, interest_count, image_url, created_at, updated_at`)으로 변경

## [2026-05-06] 댓글 목록 조회 응답에서 user_id 누락

- **파일:** `src/services/recipeCommentService.js` — `getCommentsByRecipeId`
- **원인:** SQL SELECT 절에 `rc.user_id`가 누락되어 있어 응답 객체에 포함되지 않음
- **증상:** `GET /api/recipes/:recipeId/comments` 응답의 각 댓글 객체에 `user_id`가 없어, 프론트엔드에서 '내가 쓴 댓글인지' 판별 불가 → 수정/삭제 버튼 표시 로직 동작 불가
- **수정:** SELECT에 `rc.user_id` 추가 및 응답 객체 매핑에 `user_id: Number(c.user_id)` 추가

## [2026-05-06] recipe_id가 number 아닌 string으로 직렬화되는 문제

- **파일:** `src/services/recipeService.js` — `createRecipe`
- **원인:** `recipe_id` 컬럼이 `BIGSERIAL`(64비트 정수)로 정의되어 있어, `pg` 드라이버가 JavaScript Number 정밀도 한계를 고려해 BIGINT/BIGSERIAL을 항상 string으로 직렬화함
- **증상:** `POST /api/recipes` 응답의 `recipe_id` 값이 `"6"` 처럼 따옴표로 감싸진 string으로 반환됨
- **수정:** `parseInt(row.recipe_id)`로 변환 후 반환

## [2026-05-06] BIGINT 직렬화로 recipe_id / user_id / comment_id가 string으로 반환되는 문제 (전체)

- **파일:** `src/services/recipeService.js`, `src/services/recipeCommentService.js`
- **원인:** `pg` 드라이버가 BIGINT/BIGSERIAL 컬럼을 JS Number 정밀도 한계를 고려해 string으로 직렬화함
- **증상:** 레시피 목록 조회, 상세 조회, 관심 등록/해제, 양조장 레시피 등록, 양조장 소비자 레시피 확인, 댓글 작성 API 응답에서 숫자 ID 필드가 `"6"` 형태의 string으로 반환됨
- **수정:** 각 서비스 return 지점에서 `Number()` 변환 적용

## [2026-05-06] 댓글 수정/삭제 시 본인 작성 댓글임에도 403 권한 오류 발생

- **파일:** `src/middlewares/authMiddleware.js`
- **원인:** JWT 페이로드의 `id`는 string(`"5"`)인데, DB에서 가져온 `comment.user_id`는 `Number(5)`로 변환되어 있어 `!==` 비교 시 항상 불일치 → 모든 댓글이 타인 댓글로 판정
- **증상:** `PUT /api/recipes/:recipeId/comments/:commentId`, `DELETE /api/recipes/:recipeId/comments/:commentId` 요청 시 본인 댓글도 403 반환
- **수정:** `authMiddleware`에서 JWT 디코딩 후 `req.user = { ...decoded, id: Number(decoded.id) }` 처리
