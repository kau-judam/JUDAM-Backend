### 작업 1 — AI 필터링 차단 정책 강화

**배경**: `feature/ai-law-filter` 머지 및 실제 서버 배포 후 레시피 등록 요청을 보냈을 때 AI 필터링이 작동하지 않는 것을 확인했다. 원인은 실제 서버 `.env`에 `AI_SERVER_URL`이 설정되지 않아 기본값 `localhost:8000`으로 시도하다 연결 실패 → 기존 폴백(통과 처리) 로직이 실행된 것이었다.

**변경 내용**: AI 서버와 통신이 안 되는 모든 경우에 레시피 등록을 차단하도록 정책 변경

| 상황 | 변경 전 | 변경 후 |
|------|--------|--------|
| AI 서버 다운 | 통과 (201) | **차단 (400)** |
| 5초 타임아웃 | 통과 (201) | **차단 (400)** |
| AI 서버 오류 응답 | 통과 (201) | **차단 (400)** |
| 위반 키워드 감지 | 차단 (400) | 차단 (400) |
| 정상 통과 | 통과 (201) | 통과 (201) |

**사유별 응답 메시지**:

| 상황 | message |
|------|---------|
| 법률 위반 감지 | AI 서버가 내려주는 메시지 (예: `"'어린이' 표현을 수정해주세요"`) |
| 법률 위반 (메시지 없을 때) | `"등록할 수 없는 내용이 포함되어 있습니다. 레시피 내용을 다시 확인해 주세요."` |
| 5초 타임아웃 | `"AI 서버 응답 시간이 초과되었습니다. 잠시 후 다시 시도해주세요."` |
| 연결 실패 (서버 다운 등) | `"AI 서버에 연결할 수 없습니다. 잠시 후 다시 시도해주세요."` |
| AI 서버 오류 응답 (5xx 등) | `"AI 서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요."` |

**수정 파일**: `src/utils/aiFilterInterface.js`
- `response.ok` 아닌 경우: 통과 → 차단, 오류 전용 메시지 반환
- catch 블록: `AbortError`(타임아웃) / 연결 실패 각각 다른 메시지로 분기

### 세부사항

- 실제 서버에서 AI 필터링이 동작하려면 EC2 인스턴스의 `.env`에 `AI_SERVER_URL=http://10.0.11.241:8000`을 추가하고 서버를 재시작해야 한다. 이 설정 없이는 이 브랜치 머지 후 레시피 등록이 전면 차단된다.
