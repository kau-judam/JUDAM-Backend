### 작업 1 — AI 법률 필터링 실제 연동

**이 작업이 하는 일**: 레시피 등록 시 AI 서버에 내용을 보내 법적 위반 여부를 검사한다. 위반이 감지되면 레시피 등록이 400으로 차단된다.

**기존 상태**: `src/utils/aiFilterInterface.js`가 항상 `passed: true`를 반환하는 빈 함수였음 (AI팀 연동 전 임시 처리)

**변경 내용**: AI 서버(`POST /api/law/filter`)를 실제로 호출하도록 교체

**필터링 흐름**

```
POST /api/recipes 요청
  → aiFilterInterface.checkRecipeLegalFilter() 호출
  → AI 서버 POST http://{AI_SERVER_URL}/api/law/filter 호출
  → violation: true  →  400 반환 (레시피 등록 차단)
  → violation: false →  DB 저장 후 201 반환
```

**AI 서버 요청 바디 매핑**

| 백엔드 필드 | AI 서버 필드 |
|------------|-------------|
| `title` | `title` |
| `content` | `description` |
| `main_ingredient` | `ingredients` (배열로 변환) |

**AI 서버 다운 시 폴백**: 연결 실패 또는 5초 타임아웃 발생 시 경고 로그만 남기고 통과 처리 → 서비스 중단 없음

**감지 위반 유형 (키워드 기반, API 키 불필요)**

| 유형 | 예시 키워드 |
|------|-----------|
| 미성년자 타겟 | 청소년, 어린이, 미성년자 |
| 과대광고 | 숙취없는, 건강에 좋은, 치료효과 |
| 펀딩 금융 규제 | 원금보장, 수익보장 |

### 세부사항

- AI 서버는 현재 실제 서버(43.202.24.223)에 배포되어 있지 않다. 실제 서버에서 레시피 등록 요청이 오면 AI 서버 연결 실패 → 폴백으로 통과 처리된다. AI팀이 실제 서버에 AI 서버를 배포하면 자동으로 필터링이 활성화된다.
- `AI_SERVER_URL` 환경변수로 AI 서버 주소를 관리한다. `.env.example`에 추가 완료.
